## 00:00–02:58｜開場與課程範圍

老師說明，2026 年上半年出現許多新功能與新工具，這堂一小時課程會快速回顧四個方向：文字、生圖、AI Agent 與自動化。內容不會講得很深，而是集中在最近最常受邀分享、也最實用的題目。

Agent 在下半年會繼續加速發展，因此先把上半年的基礎補齊。已經使用過相關工具的人可以比較自己的方法；還沒跟上的人則把這堂課當成趨勢與基本操作的總整理。

## 02:58–05:48｜專案資料不要只求多

老師先談 ChatGPT 專案、Gemini Gem 與 Notebook 類功能。這些已經逐漸成為大型語言模型中的日常功能，但常見問題是使用者建立大量專案，並一次丟進數十甚至數百個檔案。

老師提醒：能上傳很多檔案，不代表模型能有效消化。Word、Excel、PowerPoint 等檔案除了文字，還帶有格式與版面資訊；如果真正需要的是文字內容，讓 AI 讀取過多格式資料會增加容量、解析時間與 token 消耗。

專案應該有明確主題與用途。個人化設定像整個聊天大廳的共同規則；專案設定則是特定任務的角色、資料與工作方式。

## 05:48–09:42｜建立「颱風預測專家」與轉成 Markdown

老師以台灣颱風預測為例，先請 AI 寫出完整的專家角色指令。除了「颱風專家」這個身份，還要加入台灣脈絡、風雨、登陸機率、交通、航班、停班停課、山區與沿海風險等具體任務。

在 Canvas／文件編輯區中，可以圈選局部文字要求修改，不必讓 AI 全部重寫。例如在任務說明中補上繪製等壓線、颱風路徑、降雨圖等能力。

完成後可下載 PDF、DOCX，也可以直接要求 AI 輸出 Markdown。老師比較示範檔案大小：相同內容的 `.md` 約 9.1 KB，而帶格式的文件約 390 KB。資料一多時，Markdown 輕量、結構清楚的優勢會很明顯。

## 09:42–12:40｜專案人設、資料來源與工作流

建立專案後，老師不會立刻問工作問題，而是先到專案設定放入角色指令，再問：「你是誰？你有什麼功能？」用這個方式確認模型是否真的進入指定角色。

除了人設，還可以新增背景資料，讓回覆有可依據的來源。老師以颱風與氣象圖為例，說明可將常用資料加入專案，形成類似 RAG 的工作方式。

工作流也可以直接寫進指令，例如：先完成文字論述，再用影像模型生成一張圖佐證。這會把單次問答變成固定的「先寫文字、再做圖」流程。

## 12:40–20:53｜文件輸出、來源查證與文字篇總結

大型語言模型已能輸出 DOCX、Excel、CSV、PPT 等常見辦公格式。老師示範將找到的資料整理成 DOCX，再放回專案成為後續資料來源。

接著請颱風專家製作指定日期的天氣內容，並要求完成文字後自動畫圖。因為角色、人設、來源與工作流都已先設定，模型能用較穩定的方式產生圖文結果。

生成後仍要查證。老師建議開啟網頁搜尋，要求模型複查上述資訊，並整理成 checklist 表格，讓使用者可以逐項核對。不能因為回答列出來源，就省略反查。

文字篇的結論是：當資料量增加時，Markdown 是 AI 專案與 Notebook 很重要的朋友。它不帶多餘格式，能減少解析負擔，讓資料更容易跨工具保存與使用。

## 20:53–25:48｜Gemini Notebooks × NotebookLM 的三個提醒

Gemini Notebooks 與 NotebookLM 結合後，可以在 Gemini 介面中使用 NotebookLM 的來源資料；但老師提醒三個容易誤會的地方：

1. **共享限制**：由 Gemini Notebooks 建立或開啟的 NotebookLM 內容，當時可能無法正常共享；兩邊的共享按鈕都會受限。
2. **仍可能產生幻覺**：在 Gemini 介面問問題時，運作核心仍是 Gemini。它只是引用 NotebookLM 的資料，不代表會像 NotebookLM 原生介面一樣嚴格受限於來源。要降低幻覺，仍應在 NotebookLM 介面核對。
3. **設定可能互通**：在一邊設定的角色與語氣，可能同步到另一邊；不能假設兩邊可以設定成完全獨立的角色互相對話。

直接使用 Gemini 串 NotebookLM 上手較快，但老師認為更進階的方式是讓 AI Agent 透過 MCP 串接 NotebookLM。現在可以直接下指令請 Agent 安裝與連線，不必像早期那樣手動處理很多步驟。

## 25:48–27:09｜文字篇補充與切換主題

老師補充，角色人設最好放在特定專案，不要讓整個大廳都固定成單一角色。個人化設定可以保留較通用的原則；專門的颱風、料理或研究角色則放在各自專案中。

之後進入生圖篇。

## 27:09–34:49｜生圖模型、替代方案與文字轉影片

生圖工具很多，不需要被單一模型綁住。老師以 Nano Banana 與 ChatGPT Image 為主要比較，也提到其他模型。不同模型在真實感、文字、構圖、創意與多模態理解上各有強項；如果某一個模型額度用完，可以換模型或改變生成策略。

老師也示範工具中的文字轉影片功能：先準備一張圖，再讓工具生成約一分鐘的影片、文字與語音。免費方案通常有使用額度，若當日額度不足就要等待或改用其他工具。

這一段的重點不是追求唯一最佳模型，而是準備至少兩套可替換的工具，並依需求和額度調整。

## 34:49–37:40｜生圖不好時，先精準指出修改位置

生圖後的修正有兩派：

- 請 AI 直接修。
- 拆成獨立元素或圖層，自己修。

如果要請 AI 修，不能只說「這裡不好」。老師示範在圖片上標記區域、畫線或加註解，明確指出哪裡要增加船、魚或改變海洋顏色。送出前要保存標記，並要求 AI 完成後移除紅線與註解。

這種局部標註比純文字描述更清楚；在手機上也可以先截圖畫記號再交給 AI。

## 37:40–41:43｜批次生成、逐張生成與拆解圖片

部分付費影像模型可以一次產生多張圖。如果沒有相同方案，也可以要求模型「依序生成」不同版本。

老師將一張含多個颱風角色與台灣圖案的圖片拆解，要求：

- 每個角色重新畫成獨立圖片。
- 保持與原圖一致。
- 統一 16:9。
- 去除背景。

這種方法可以把一張資訊圖或簡報封面拆成多個可重用素材。若要製作整份簡報，也可以先定義共同風格，再依序生成封面、目錄與內容圖片。

## 41:43–47:21｜Canva 魔法圖層與可編修素材

另一種拆解方式是串接 Canva，使用魔法圖層把圖片中的文字、人物與裝飾元素拆開。課堂示範中，ChatGPT 透過工具連線叫出 Canva，再要求魔法圖層解構圖片。

魔法圖層一次通常處理一張；如果要處理多頁，可以在 Canva AI 中下指令批次處理。拆解結果可能有部分文字或物件不完整，因此可以多試幾次，再下載成 PPT 做後續編輯。

老師個人的習慣是借用 Canva 的拆圖層能力，之後回到 PowerPoint 調整；學生則常直接留在 Canva 編排。重點是把圖片變回可控制的物件，而不是接受一張完全扁平、無法修改的圖。

## 47:21–50:42｜AI Agent、Codex 與 NotebookLM MCP

Agent 的差異是它能把多個步驟連續完成，而不必像一般聊天一樣每一步都由使用者來回操作。老師以 Codex 為代表，說明 Agent 通常需要下載或在專用環境中使用，也會顯示額度與期限。

早期常見玩法是替 Agent 加入 MCP 工具。現在可以直接下指令：「請幫我連上 NotebookLM 的 MCP」，Agent 會尋找並安裝連線工具。

實際串接仍需要登入 Google。課堂示範中，Agent 開啟 Google 登入頁，提醒使用者必須完成帳號授權後才能讀取 NotebookLM。

## 50:42–53:21｜Computer Use、瀏覽器、GitHub 與完整任務

老師常使用的 Agent 能力包括：

- Computer Use：操作電腦介面。
- Browser Use：瀏覽與操作網頁。
- GitHub：尋找程式、讀取專案或配合開發流程。
- MCP 與 Skills：加入新的工具及可重複能力。

示範任務是：查詢台灣下週官方天氣與颱風假判斷標準，完成 SWOT 分析，加入圖表並輸出 Word 報告。這種任務如果放在一般聊天中，使用者需要一問一答、來回切換；Agent 則可以自己搜尋、整理、製圖與輸出文件。

## 53:21–55:21｜多 Agent 分工與 Codex 手機連線

老師平常會讓 Codex、OpenCode、Antigravity、Claude Code 等工具各自負責不同工作。若要讓多個 Agent 一起做事，可以讓它們在同一個專案資料夾工作，並事先分配角色，例如：

- Gemini／Antigravity：前端視覺與渲染。
- Claude Code：程式引擎或長脈絡開發。
- GPT：圖片生成。
- OpenCode：資料搜尋或其他終端任務。

Codex 也能透過 ChatGPT App 與手機連線。連線後，可以在外面用手機下指令，電腦端的 Codex 繼續執行工作。

多 Agent 很有趣，但也會增加成本；應保留給重要、複雜且值得分工的專案。

## 55:21–57:02｜生圖補充與影片工具

老師回到生圖結果，補充如果沒有 Canva，也可以讓影像模型把圖片逐一拆解，並輸出可在簡報中重用的素材。之後播放前面生成的短影片，說明影片包含聲音，素材也能下載到 Google Vids 之類的工具繼續調整。

## 57:02–59:02｜自動化未完整講授與課程結束

老師明確說明剩下的自動化內容來不及講，OpenCode、Claude Code 等工具也沒有逐一深入介紹。後續可以另開課程補充 Agent、Skills 與安裝使用。

最後提到 Antigravity 2.0 的 App／中文版，以及 Codex、OpenCode 等工具在介面與使用方式上有相似之處，然後結束直播。
